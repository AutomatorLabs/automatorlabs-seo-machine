## Purpose

- Keep a concise, repo-local source of truth.
- Capture how the Astro site is organized.
- Document current calculators, guides, and SEO systems.
- Reduce repeated codebase scanning in future Codex sessions.

## For AI Agents

- Read this folder before proposing implementation work.
- Prefer these docs for orientation, then verify details in source.
- Treat any line marked `Needs confirmation` as unresolved.
- Update these docs after meaningful product, architecture, or SEO changes.

## For Humans

- Keep entries short, factual, and tied to current code.
- Do not describe planned features as live behavior.
- When uncertain, write `Needs confirmation`.
- Update docs in the same PR/session as the underlying change whenever possible.

## Maintenance Rules

- Keep documentation inside `/docs`.
- Favor checklists and short bullets over long prose.
- Preserve file names so future agents can rely on stable entry points.
- Add dated notes to [CHANGELOG.md](/Users/tim/Desktop/MICRO%20SAAS/automatorlabs-seo-machine/docs/CHANGELOG.md) when this knowledge base changes materially.
